## Фича: импорт «без перемещения файлов»

Добавляем в панель импорта (режимы Фото/Видео) галочку «Не перемещать файлы». Когда она включена, файлы остаются по своему исходному пути, а в библиотеку записывается только путь к оригиналу + сгенерированная миниатюра (всё ещё внутри папки приложения). Новый булев флаг на элементе — `external`.

### 1. main.js — модель данных
- **`normalizeMetadata`** (~106-135): добавить приведение `external` к строгому boolean по образцу `favorite` (строки `"true"`/`1` → `true`), с `changed=true`. Так старые metadata.json автоматически добиваются полем при чтении.

### 2. main.js — импорт без перемещения
- **`move-file` IPC** (~1094-1120): добавить 4-й параметр `external`. Если `true` — пропустить `rename`/`copy`+`unlink`, в качестве `path` вернуть исходный `srcPath`, но по-прежнему вызвать `generateThumbnail(srcPath, mediaType)` и (для видео) `getVideoDuration(srcPath)` и вернуть их. Миниатюра по-прежнему пишется в `DATA_ROOT/.../thumbnails`.

### 3. main.js — безопасное удаление оригинала
- **`removeMediaFiles`** (~762-770): если `item.external` — НЕ удалять `item.path` (фото/видео) и не сносить comic-папку. Сгенерированная миниатюра фото/видео (лежит в `DATA_ROOT`) удаляется как обычно. Это покрывает и IPC `delete-media` (1342), и cast `delete-media` (917), т.к. обе идут через эту функцию.

### 4. main.js — стрим внешних файлов на телефон (cast-сервер)
- **`/media` handler** (~945-954): сейчас отдаёт только то, что внутри `DATA_ROOT` (`isPathInside`). Расширить: если путь снаружи — отдавать его, **только если** он числится в метаданных как `path`/`thumbnail`/`pages` какого-либо элемента с `external:true`. Произвольные пути (системные файлы и т.п.) остаются 403.
- Чтобы не читать metadata.json при каждом запросе картинки — кэш разрешённых внешних путей на уровне модуля, инвалидация по `mtime` файла метаданных (`fs.statSync` → при изменении перестроить множество). Чтение остальных данных метаданных не трогает.

### 5. preload.js
- **`moveFile`** (~10): добавить 4-й параметр `external` → `ipcRenderer.invoke('move-file', srcPath, category, mediaType, external)`.

### 6. renderer/index.html
- В секцию «Импорт» (после `<select id="import-category">`, ~78) добавить `<label class="toggle-line"><input type="checkbox" id="import-no-move"><span>Не перемещать файлы</span></label>` (использую существующий стиль `toggle-line`, как у «Исключить теги»).

### 7. renderer/app.js
- **`importPaths`** (~689-715): принимать флаг `external`, передавать в `window.api.moveFile(path, category, state.mode, external)` и в `addMediaEntry`.
- **`addMediaEntry`** (~757-774): принимать `external`, ставить `external: !!external` в объекте item.
- **`importMove`** (~536-550) и **`selectFiles`** (~552-560): читать `$('#import-no-move').checked` и передавать в `importPaths`.
- **`updateImportMode`** (~266-274): скрывать чекбокс в режиме комиксов (комиксы всегда копируются в папку приложения), показывать в фото/видео — по аналогии с `#import-move`.
- **Тексты подтверждения удаления** (`deleteMedia` ~1613, bulk delete ~1125): адаптировать — при external-элементах говорить «удалить из библиотеки» (файл останется), чтобы не вводить в заблуждение. Мелкая правка формулировок.

### 8. Состояние галочки
- Состояние UI, по умолчанию выкл. Для удобства сохранять/восстанавливать из `localStorage` (как тему) — необязательно, но дёшево.

### Поведение
- **Фото/Видео, галочка вкл:** файл остаётся на месте; в библиотеке путь = оригинал, `external:true`, миниатюра своя. В окне приложения виден через `file://`. На телефоне (трансляция) — стримится через `/media` с проверкой по метаданным.
- **Фото/Видео, галочка выкл:** текущее поведение (перемещение в `DATA_ROOT`).
- **Комиксы:** не затронуты — всегда копируются в папку приложения; галочка скрыта.
- **Удаление external-элемента:** удаляется только запись + миниатюра, оригинал нетронут.

### Что НЕ меняется
`generateThumbnail`, `getVideoDuration`, `saveVideoThumbnailFromInput`, select/paste превью видео, `lookupVisibleVideoDurations`, просмотрщик — все работают с любым абсолютным путём без правок.

Файлы для изменения: `main.js`, `preload.js`, `renderer/index.html`, `renderer/app.js`. CSS правок не нужно (переиспользую `toggle-line`).
